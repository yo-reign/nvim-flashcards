# nvim-flashcards v1 Ground-Up Rewrite

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Full rewrite of nvim-flashcards with manual-only tags, named tag scopes, configurable JSON/SQLite storage, soft-delete orphans, and production-quality code.

**Architecture:** Modular Lua plugin with explicit state-machine parser, storage interface pattern (JSON + SQLite backends), FSRS binary scheduler, nui.nvim review UI, and Telescope pickers. Cards identified by inline IDs, plugin force-reloads buffers after write-back.

**Tech Stack:** Lua 5.1 (LuaJIT), Neovim API, plenary.nvim (async/paths/testing), nui.nvim (UI), telescope.nvim (pickers), sqlite.lua (optional backend)

## Syntax Reference (for all tasks)

```markdown
<!-- Inline cards -->
front ::: back #tag1 #tag2 <!-- fc:abc12345 -->
front :?: back #tag1 <!-- fc:abc12345 -->

<!-- Fenced cards -->
:::card <!-- fc:abc12345 -->
front content
:-:
back content
:::end #tag1 #tag2

:?:card <!-- fc:abc12345 -->
front content
:-:
back content
:?:end #tag1

<!-- Tag scopes (named close, nestable) -->
:#python:
:#decorators:
cards here get both #python and #decorators
:#/decorators:
cards here get only #python
:#/python:

<!-- Source annotations -->
<!-- note: Serge Lang Ch.1 p.12 -->

<!-- Suspended cards -->
front ::: back <!-- fc:abc12345 !suspended -->

<!-- Template variables (expand at parse time) -->
:#{{file.dir}}/{{file.name}}:
<!-- note: {{file.name}} (1.2.3:5) -->
```

**Step 3: Implement parser.lua**

State machine with three states:

```
NORMAL -> FENCED_FRONT (on :::card or :?:card)
FENCED_FRONT -> FENCED_BACK (on :-:)
FENCED_BACK -> NORMAL (on :::end or :?:end)
```

Additional tracking:
- `code_block_depth` in NORMAL state (skip content inside ```)
- `fenced_code_depth` inside fenced cards (don't match :::end inside ```)
- `scope_stack` for named tag scopes with line numbers

Public API:
- `M.parse(file_path, content, scan_root)` -> `cards[], errors[]`

Each card is a table:
```lua
{
  front = "...",
  back = "...",
  reversible = false,
  id = "abc12345" or nil,
  suspended = false,
  tags = { "math", "algebra" },
  note = "Ch.1 p.5" or nil,
  line = 42,
  file_path = "math/algebra.md",
}
```

ID write-back:
1. Parse file, identify cards without IDs
2. Sort by line number descending (bottom-up)
3. For each: generate ID, append `<!-- fc:ID -->` to the line
4. Write file back
5. Call `vim.api.nvim_buf_call(bufnr, function() vim.cmd("checktime") end)` for open buffers

Orphan detection:
- Per-file: after scanning a file, check if any stored cards for that file_path are no longer in parsed results -> mark lost
- Global: after scanning all dirs, check if any active cards have IDs not found in any scanned file -> mark lost
